### 1. Via 浏览器获取网页 Cookie 的方法

**适用场景：** 在手机 Via 浏览器中获取当前网页的 Cookie

**操作步骤：**
1. 在 Via 浏览器中打开目标网页
2. 点击地址栏，删除原网址
3. 输入以下代码：
   ```
   javascript:alert(document.cookie)
   ```
4. 按前往/确认
5. 弹窗中显示的内容即为当前网页的 Cookie

**注意事项：**
- 若弹窗为空，说明网站设置了 HttpOnly（禁止 JS 读取），需使用抓包工具
- 复制 Cookie 时要完整，包括所有 `key=value` 对，用分号 `;` 隔开

**记录时间：** 2026-03-16
**首次成功：** 主人通过此方法成功获取了目标网页 Cookie

### 4. 🚨 飞书文档操作重要教训（三次犯错记录）

**时间：** 2026-03-31  
**严重程度：** 🔴 极高 - 导致用户三次反馈错误

---

#### 错误一：文档内容为空

**问题描述：**
Cron 任务创建飞书文档后，文档内容为空（只有标题）

**问题原因：**
Cron 任务的指令只要求"将所有信息整理成一个飞书文档"，但执行的 agent 只创建了文档，**没有调用 `feishu_doc write` 写入内容**

**解决方案：**
在 Cron 任务配置中添加明确的步骤指令：
```
**重要步骤**：
1. 使用 feishu_doc create 创建飞书文档
2. 使用 feishu_doc write 将完整内容写入文档（这一步必须执行！）
3. 获取文档链接
4. 发送飞书卡片消息，附上文档链接
```

---

#### 错误二：缺少新闻来源链接

**问题描述：**
生成的飞书文档内容中，每条新闻缺少来源链接

**问题原因：**
- Cron 任务指令中要求"每条新闻必须附上来源和链接"
- 但执行时只是从摘要中复制内容，**没有保留完整的链接**
- 用户反馈后需要重新搜索获取完整链接

**解决方案：**
Cron 任务指令中明确要求保留链接：
```
输出要求：
- 每条新闻/信息必须附上来源和链接
- 格式：📰 [新闻标题] - [一句话摘要]
        🔗 来源：[网站名称] | [完整链接]
- 如果某条信息无法找到确切来源，标注"来源待确认"
```

**执行时必须：**
1. 保留搜索结果中的完整 URL
2. 不要只复制摘要，要包含来源信息
3. 检查每条新闻是否都有链接

---

#### 错误三：使用百度跳转链接而非真正的源链接

**问题描述：**
文档中的新闻链接显示为 `http://www.baidu.com/link...`，用户无法访问

**问题原因：**
- 百度搜索结果中的链接是**跳转链接**，不是真正的源 URL
- 格式：`http://www.baidu.com/link?url=xxx`
- 这是百度的跳转服务器，直接访问可能失败

**解决方案：**
1. **优先使用 web_search (Tavily)** - 返回的链接是真实 URL
2. **如果必须用百度** - 需要用 web_fetch 访问跳转链接获取真实 URL
3. **验证链接格式** - 确保是可直接访问的完整域名（如 `https://www.reuters.com/...`）

**示例：**
```
❌ 错误：🔗 来源：半导体行业观察 | http://www.baidu.com/link?url=xxx
✅ 正确：🔗 来源：Reuters | https://www.reuters.com/world/asia-pacific/chinas-no-2-chipmaker-readies-7-nm-production-2026-03-16/
```
