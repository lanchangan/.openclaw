# 经验教训记录

## 2026-03-30 早报任务问题总结

### 问题1：模型 content_filter 导致内容生成失败

**现象**：执行早报任务时，minimax 和 glm-5 都返回 "你好，我无法给到相关内容"

**原因**：
- minimax：触发了内容过滤器 (content_filter)
- glm-5：第一次运行时也返回了同样的问题（原因待查）

**解决**：使用 glm-5 重新运行后成功生成内容

**预防**：
- 涉及金融、股票内容的任务避免使用 minimax
- 测试时如遇问题，可尝试重新运行
- 考虑使用 kimi 或其他模型作为备选

---

### 问题2：feishu_doc create 操作 bug

**现象**：使用 `feishu_doc action=create` 创建文档时，只创建了标题，内容为空（block_count=1）

**原因**：feishu_doc 工具的 create 操作存在 bug，content 参数没有正确写入

**解决**：创建文档后，使用 `feishu_doc action=write` 重新写入内容

**正确流程**：
1. 先 create 创建文档（获取 doc_token）
2. 再使用 write 写入完整内容

---

### 问题3：飞书文档 Markdown 链接无法点击

**现象**：使用 `[文本](URL)` Markdown 格式写入链接，在飞书文档中显示为纯文本，无法点击

**原因**：飞书文档 API 对 Markdown 链接支持有限，不会自动转换为超链接

**解决**：改为直接显示完整 URL，如 `https://example.com`

**预防**：
- 飞书文档中避免使用 Markdown 链接语法
- 如需可点击链接，考虑使用飞书多维表格或其他富文本编辑器

---

### 总结

执行自动化任务时，应该：
1. **先测试模型**是否支持任务内容类型
2. **验证文档创建**是否成功写入内容
3. **检查链接格式**在目标平台是否有效
