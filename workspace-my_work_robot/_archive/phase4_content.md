### Day 36: 空间复用

#### 核心概念

**1. MIMO基础**

```
MIMO = Multiple-Input Multiple-Output

核心思想：
- 使用多个发射天线和多个接收天线
- 在相同带宽上并行传输多路数据流
- 利用空间维度提升容量

SISO vs MIMO：
┌─────────────────────────────────────────────┐
│ SISO: 1发1收，单数据流                       │
│   TX ──→ RX                                 │
│   容量 ≈ B·log₂(1+SNR)                      │
│                                             │
│ MIMO: N发M收，多数据流                       │
│   TX₁ ──↘                                   │
│   TX₂ ──→ RX₁, RX₂, ..., RX_M              │
│   ...  ↗                                    │
│   容量 ≈ N·B·log₂(1+SNR/N)  (理想情况)      │
└─────────────────────────────────────────────┘
```

**2. 空间复用原理**

```
信号模型：
y = H·x + n

其中：
- y ∈ C^(M×1): 接收信号向量
- H ∈ C^(M×N): 信道矩阵
- x ∈ C^(N×1): 发送信号向量
- n ∈ C^(M×1): 噪声向量

MIMO信道容量：
C = max log₂(det(I + SNR·H·H^H))

高SNR近似：
C ≈ min(N,M)·log₂(SNR)

即容量与min(发射天线数,接收天线数)成正比
```

**3. 信道矩阵特性**

```
信道矩阵H的奇异值分解（SVD）：
H = U·Σ·V^H

其中：
- U: 左奇异向量（接收端）
- Σ: 奇异值对角矩阵
- V: 右奇异向量（发送端）

奇异值含义：
- 每个奇异值对应一个空间"管道"
- 大奇异值 → 强空间路径
- 小奇异值 → 弱空间路径

条件数：
κ(H) = σ_max/σ_min

κ大 → 信道相关性高，空间复用增益低
κ小 → 信道独立性好，空间复用增益高
```

```
- 实际增益取决于信道条件
```

#### MIMO信道估计

```python
import numpy as np

def mimo_channel_estimate(rx_ltf, tx_ltf, N_tx, N_rx):
    """MIMO信道估计"""
    # N_tx个发送天线，N_rx个接收天线
    H = np.zeros((N_rx, N_tx), dtype=complex)
    
    for tx in range(N_tx):
        for rx in range(N_rx):
            # 对每对天线进行LS估计
            H[rx, tx] = rx_ltf[rx] / tx_ltf[tx]
    
    return H

def svd_precoding(H):
    """基于SVD的预编码"""
    U, S, Vh = np.linalg.svd(H)
    
    # Vh^H 作为预编码矩阵
    W = Vh.conj().T
    
    # U^H 作为解码矩阵
    D = U.conj().T
    
    return W, D, S

# 示例
H = np.random.randn(4, 4) + 1j*np.random.randn(4, 4)  # 4x4 MIMO
W, D, S = svd_precoding(H)

print("奇异值:", S)
print("空间流增益:", S**2)
```


**3. 预编码矩阵**

```
预编码矩阵计算：

目标：找到W使接收信号质量最优

MMSE预编码：
W = H^H·(H·H^H + σ²·I)^(-1)

ZF预编码：
W = H^H·(H·H^H)^(-1)

其中H^H表示共轭转置

预编码效果：
- 抑制流间干扰
- 提高接收信噪比
- 支持多用户传输
```

### Day 39: MU-MIMO

#### 核心概念

**1. MU-MIMO原理**

```
MU-MIMO = Multi-User MIMO

核心思想：
同时向多个用户传输数据，共享空间资源

vs SU-MIMO：
- SU-MIMO：单用户，多数据流
- MU-MIMO：多用户，每用户可能单流或多流

┌─────────────────────────────────────────────────────────┐
│            SU-MIMO vs MU-MIMO                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  SU-MIMO:                    MU-MIMO:                   │
│  ┌─────┐                     ┌─────┐                    │
│  │ AP  │───→ 用户1           │ AP  │───→ 用户1         │
│  │     │    (多流)           │     │                   │
│  └─────┘                     │     ├───→ 用户2         │
│                              │     │                   │
│                              │     ├───→ 用户3         │
│                              └─────┘                   │
│                                                         │
│  用户独占时间/频率           多用户同时传输             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**2. 下行MU-MIMO**

```
DL MU-MIMO（下行）：

信号模型：
y_k = H_k·W·s + n_k

其中：
- k = 用户索引
- W = 预编码矩阵（所有用户联合设计）
- s = 所有用户数据的堆叠

预编码设计：
目标：消除用户间干扰

块对角化：
W = [W₁, W₂, ..., W_K]

使H_i·W_j ≈ 0 (i≠j)

即：用户i不受用户j的数据干扰
```

**3. 上行MU-MIMO**

```
UL MU-MIMO（上行）：

多用户同时发送，AP联合解码

信号模型：
y = H₁·x₁ + H₂·x₂ + ... + H_K·x_K + n

AP需要：
1. 信道估计（各用户的CSI）
2. 联合检测（分离各用户信号）

检测方法：
- ZF检测：W = (H^H·H)^(-1)·H^H
- MMSE检测：W = (H^H·H + σ²I)^(-1)·H^H
- MLD：最大似然检测（最优但复杂）
```

#### WiFi MU-MIMO实现

```
┌─────────────────────────────────────────────────────────┐
│          WiFi MU-MIMO 能力                               │
├───────────────┬────────────┬────────────┬───────────────┤
│ 标准          │ DL MU-MIMO │ UL MU-MIMO │ 最大用户数    │
├───────────────┼────────────┼────────────┼───────────────┤
│ 802.11ac      │ 支持       │ 不支持     │ 4             │
│ 802.11ax      │ 支持       │ 支持       │ 8             │
│ 802.11be      │ 支持       │ 支持       │ 16            │
└───────────────┴────────────┴────────────┴───────────────┘

MU-MIMO调度：
- AP决定哪些用户同时传输
- 需要用户组管理
- 需要定期信道探测
```
