# WiFi 接收物理层时域 AGC 模块讲解

## 🎯 AGC 模块核心定位

AGC（Automatic Gain Control，自动增益控制）是接收链路的"第一道关卡"——在 LNA 之后、ADC 之前，负责动态调整前端增益，让输入信号适配 ADC 的动态范围。

### 为什么 AGC 如此关键？

| 问题 | 后果 |
|------|------|
| 增益过高 | ADC 饱和 → 信号削峰 → EVM 恶化 |
| 增益过低 | 量化噪声占主导 → SNR 损失 → 灵敏度下降 |
| 增益调整不及时 | 前导码期间收敛失败 → 无法解调 |

**一句话：AGC 决定了接收机能不能"看见"信号。**

---

## 🔧 时域 AGC 工作流程

```
天线 → LNA → [AGC控制] → VGA/可变衰减器 → ADC → 后续处理
                     ↑
                 功率检测反馈
```

### 典型时序（WiFi 6/7）

```
┌─────────────────────────────────────────────────────────┐
│  L-STF    │  L-LTF  │  L-SIG  │  ... 数据符号 ...      │
├───────────┼─────────┼─────────┼─────────────────────────┤
│ AGC粗调   │ AGC精调 │ 稳定    │ 固定增益                 │
│ (前4μs)   │ (后4μs) │         │                          │
└─────────────────────────────────────────────────────────┘
```

**关键时间窗：**
- **粗调阶段**：检测 L-STF 能量，快速估计接收功率范围
- **精调阶段**：利用 L-LTF 做更精确的功率估计，微调增益
- **锁定时刻**：在数据符号开始前必须完成增益锁定

## 📐 核心算法逻辑

### 1. 功率估计（时域）

```verilog
// 典型的滑动窗口功率估计
power_estimate = Σ |r[n]|² / N  // N = 窗口长度

// 实际实现常用简化（避免除法）
power_estimate = Σ |r[n]|² >> log2(N)
```

**验证关注点：**
- 窗口长度选择：太短 → 估计方差大；太长 → 跟踪慢
- 溢出保护：累加器位宽要够
- 时钟域：采样率 vs AGC 处理速率

### 2. 增益调整策略

```
目标：让 ADC 输入功率落在 P_target ± ΔP 范围内

          P_target - ΔP         P_target         P_target + ΔP
               │                    │                    │
    ───────────┼────────────────────┼────────────────────┼───────────
               │   ✅ 目标区间       │                    │
               │                    │                    │
         减增益 ←───────────────────┼───────────────→ 加增益
```

**增益步进策略（dB）：**
- 粗调：大步进（6-12 dB），快速逼近
- 精调：小步进（1-3 dB），精细对齐
- 迟滞控制：避免在边界频繁切换

### 3. 多天线处理（MIMO）

```
对于 N_rx 天线系统：

方案A：独立 AGC —— 每路天线独立控制
方案B：联合 AGC —— 取最大/平均功率控制统一增益

WiFi 6/7 通常采用：独立粗调 + 联合精调
```

## 🧪 验证工程师视角：验证要点

### 功能验证清单

| 场景 | 验证目标 | 覆盖要点 |
|------|---------|---------|
| 灵敏度附近 | AGC 能否将信号拉入 ADC 动态范围 | 低信噪比、小信号 |
| 大信号阻塞 | 防止 ADC 饱和 | 高增益初值、快速降增益 |
| 远近效应 | 快速跟踪功率跳变 | 帧间功率变化 40+ dB |
| 多天线一致 | 各链路增益一致性 | 天线间增益差 < 阈值 |
| 干扰存在 | 干扰下的功率估计准确性 | 窄带干扰、宽带干扰 |

### 性能指标验证

```
关键指标：
├── 收敛时间：< 符号周期（通常 < 8μs for WiFi 6）
├── 收敛精度：目标功率误差 < 1-2 dB
├── 增益切换速度：响应延迟 < 几个采样周期
└── 稳定性：收敛后增益抖动 < 阈值
```

## ⚠️ 常见 Pitfall 和调试经验

### 1. **功率估计偏差**
- **现象**：AGC 稳定后的功率与预期不一致
- **排查**：检查量化误差、窗口长度、数据类型截断

### 2. **增益震荡**
- **现象**：增益在两个档位间反复切换
- **排查**：迟滞区间设置、判决阈值

### 3. **收敛失败**
- **现象**：前导码结束仍未锁定
- **排查**：增益步进是否够大、初始增益设置

### 4. **多天线不一致**
- **现象**：各天线增益差异导致 MIMO 性能下降
- **排查**：各链路 AGC 参数一致性、校准流程

|--------|---------|

## 🔗 相关模块联动

AGC 与其他模块的接口关系：

```
                    ┌──────────────┐
                    │   天线/RF    │
                    └──────┬───────┘
                           ↓
                    ┌──────────────┐
                    │     LNA      │
                    └──────┬───────┘
                           ↓
           ┌───────────────┴───────────────┐
           │           AGC 模块            │
           │  ┌─────────────────────────┐ │
           │  │  功率检测 → 增益判决     │ │
           │  └─────────────────────────┘ │
           └───────────────┬───────────────┘
                           ↓
                    ┌──────────────┐
                    │  VGA/衰减器  │
                    └──────┬───────┘
                           ↓
                    ┌──────────────┐
                    │     ADC      │
                    └──────┬───────┘
                           ↓
              ┌────────────┴────────────┐
              ↓                          ↓
       ┌──────────────┐          ┌──────────────┐
       │   同步模块   │          │  信道估计    │
       └──────────────┘          └──────────────┘
```

**关键接口信号：**
- `agc_gain_value`: 当前增益值
- `agc_locked`: 增益锁定指示
- `agc_power_est`: 功率估计值（用于后续 SNR 估计）
- `agc_update`: 增益更新脉冲
