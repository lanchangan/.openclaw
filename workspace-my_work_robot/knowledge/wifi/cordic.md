# CORDIC 算法

> 坐标旋转数字计算机 - 高效的三角函数计算方法

---

## 一句话总结

> **CORDIC 通过多次小角度旋转逐步逼近目标角度，每次迭代只需要移位和加法，硬件实现极其高效。**

---

## 核心原理

### 类比：二分查找

```
想象你要旋转一个向量到 40° 位置：

第1次：旋转 45°（转多了）
第2次：往回转 26.565°（不够了）
第3次：再往前转 14.036°（又超了）
第4次：往回转 7.125°（更接近了）
...

经过 16 次迭代，误差 < 0.001°
```

### 关键设计

每次旋转的角度是预先设计好的：**atan(2^-i)**

| 迭代次数 | 旋转角度 | 为什么选这个角度？ |
|---------|---------|------------------|
| 第 1 次 | 45.0° | atan(2^0) |
| 第 2 次 | 26.565° | atan(2^-1) |
| 第 3 次 | 14.036° | atan(2^-2) |
| 第 4 次 | 7.125° | atan(2^-3) |
| ... | ... | ... |

**核心优势**：tan(angle) = 2^-i，所以乘法变成**移位**！

## 算法伪代码

```c
// 计算 sin(θ) 和 cos(θ)
// 输入：目标角度 θ
// 输出：cos(θ) 在 x，sin(θ) 在 y

x = 0.60725;  // 预补偿常数（所有 cos 的乘积）
y = 0;        // 初始 y 分量
z = θ;        // 剩余要旋转的角度

for (i = 0; i < N; i++) {  // N = 迭代次数
    if (z >= 0) {
        // 逆时针旋转
        x_new = x - (y >> i);   // y >> i = y / 2^i
        y_new = y + (x >> i);
        z = z - atan_table[i];
    } else {
        // 顺时针旋转
        x_new = x + (y >> i);
        y_new = y - (x >> i);
        z = z + atan_table[i];
    }
    x = x_new;
    y = y_new;
}

// 最终：x ≈ cos(θ)，y ≈ sin(θ)
```

### 核心操作

只有 3 种操作：
1. **移位**：`y >> i`（硬件极快）
2. **加法**：`x + (y >> i)`
3. **查表**：`atan_table[i]`（只有 N 个值，很小的表）

## 迭代次数与精度

### 角度分辨率

| 迭代次数 | 最小旋转角度 | 相位精度 |
|---------|-------------|---------|
| 8 次 | ~0.447° | ~0.447° |
| 12 次 | ~0.028° | ~0.028° |
| 16 次 | ~0.0017° | ~0.0017° |
| 20 次 | ~0.0001° | ~0.0001° |

### 迭代次数 vs EVM

| 目标 EVM | 需要的相位精度 | 推荐迭代次数 |
|----------|---------------|-------------|
| -30 dB | ~1° | 8 次 |
| -40 dB | ~0.3° | 12 次 |
| -50 dB | ~0.1° | 14-16 次 |
| -60 dB | ~0.03° | 18-20 次 |

## WiFi 中的应用

### 应用场景

| 场景 | 作用 |
|------|------|
| **相位旋转** | 调制/解调时的相位补偿 |
| **频偏校正** | 载波频偏引起的相位旋转校正 |
| **相位噪声补偿** | 补偿本振相位抖动 |
| **FFT 旋转因子** | e^(-j2πkn/N) 的计算 |

### 为什么用 CORDIC 而不是查表？

| 方法 | 优点 | 缺点 |
|------|------|------|
| **大查表法** | 快 | 存储量大，精度有限 |
| **CORDIC** | 硬件简单，精度可调 | 需要迭代，有延迟 |

### CORDIC 优势

- 只需要移位器和加法器（FPGA/ASIC 友好）
- 精度通过迭代次数控制
- 不需要乘法器（或只需要少量）

## 硬件实现考量

### 资源消耗

```
N 次迭代需要：
- N 个移位器（或 1 个可配置移位器，N 个周期）
- 2N 个加法器（或时分复用）
- 1 个小查表（N 个 atan 值）
```

### 流水线设计

```
迭代 1 → 迭代 2 → 迭代 3 → ... → 迭代 N
   ↓         ↓         ↓              ↓
  输入     中间结果   中间结果       输出

流水线化后，每个周期可输出一个结果
```

### 验证关注点

|--------|------|
| **精度验证** | 不同迭代次数下的输出精度 |

| **时序验证** | 流水线时序是否正确 |
