## 1. DCM 概述

### 1.1 什么是 DCM

DCM (Dual Carrier Modulation) 双载波调制是一种频率分集技术，将同一数据在两个子载波上重复传输。

**核心思想**：
- 将数据复制到两个子载波
- 如果一个子载波受干扰，另一个仍可解调
- 牺牲速率换取可靠性

### 1.2 DCM 在 WiFi 6 中的应用

```
WiFi 6 DCM 支持:
- 仅用于特定 MCS
- 主要用于低速率、高可靠性场景
- MCS 0, 1, 3, 4 可选支持 DCM

DCM vs 非 DCM:
┌─────────────────┬────────────┬────────────┐
│     参数        │  非 DCM    │    DCM     │
├─────────────────┼────────────┼────────────┤
│ 数据速率        │    1x      │   0.5x     │
│ 频率分集        │    无      │   2 倍     │
│ 可靠性          │   标准     │   更高     │
│ 适用场景        │ 正常通信   │ 干扰环境   │
└─────────────────┴────────────┴────────────┘
```

## 2. DCM 原理

### 2.1 子载波映射

```
非 DCM 模式:
子载波:  1   2   3   4   5   6   7   8
数据:   d1  d2  d3  d4  d5  d6  d7  d8

DCM 模式:
子载波:  1   2   3   4   5   6   7   8
数据:   d1  d1  d2  d2  d3  d3  d4  d4
         └──┘   └──┘   └──┘   └──┘
          ↑      ↑      ↑      ↑
        重复   重复   重复   重复
```

### 2.2 DCM 映射规则

```
DCM 映射:
- 将 N 个数据符号映射到 2N 个子载波
- 每个数据符号在两个不同子载波上重复
- 两个子载波频率间隔较大（频率分集）

示例 (26-tone RU):
数据符号: 20 个
子载波:   24 个数据子载波 + 2 个导频

DCM 映射:
- 数据符号映射到 20 对子载波
- 每对子载波相距至少 10 个子载波间隔
```

### 2.3 DCM 与调制

```
DCM 支持的调制:
- BPSK: 可用 DCM
- QPSK: 可用 DCM  
- 16-QAM: 可用 DCM
- 64-QAM 及以上: 不支持 DCM

DCM-MCS 映射:
┌───────┬─────────┬──────────────────┐
│  MCS  │  调制   │    DCM 支持      │
├───────┼─────────┼──────────────────┤
│  0    │  BPSK   │    可选          │
│  1    │  QPSK   │    可选          │
│  2    │  QPSK   │    否            │
│  3    │  16-QAM │    可选          │
│  4    │  16-QAM │    可选          │
│  5+   │  64+QAM │    否            │
└───────┴─────────┴──────────────────┘
```

## 3. DCM 实现

### 3.1 发射机处理

```
发射机 DCM 流程:

1. 扰码 (Scrambling)
   └─→ 加扰比特流

2. 编码 (Encoding)
   └─→ BCC 或 LDPC 编码

3. 交织 (Interleaving)
   └─→ 分散突发错误

4. 调制 (Modulation)
   └─→ BPSK/QPSK/16-QAM 星座映射

5. DCM 映射 ←── 关键步骤
   └─→ 每个符号复制到两个子载波

6. 导频插入
   └─→ 添加导频子载波

7. OFDM 调制 (IFFT)
   └─→ 时域信号

8. 加循环前缀
   └─→ 最终发射信号
```

### 3.2 DCM 映射示例

```python
# DCM 映射伪代码
def dcm_mapping(data_symbols, num_subcarriers):
    """
    将数据符号映射到子载波（DCM模式）
    """
    num_pairs = len(data_symbols)
    subcarriers = [0] * (num_pairs * 2)
    
    for i, symbol in enumerate(data_symbols):
        # 第一子载波
        subcarrier_idx_1 = get_dcm_subcarrier_1(i, num_subcarriers)
        subcarriers[subcarrier_idx_1] = symbol
        
        # 第二子载波（重复）
        subcarrier_idx_2 = get_dcm_subcarrier_2(i, num_subcarriers)
        subcarriers[subcarrier_idx_2] = symbol
    
    return subcarriers

def get_dcm_subcarrier_1(i, N):
    """获取 DCM 第一子载波索引"""
    return i

def get_dcm_subcarrier_2(i, N):
    """获取 DCM 第二子载波索引（频率分集）"""
    return i + N // 2  # 相隔半带宽
```

### 3.3 接收机处理

```
接收机 DCM 流程:

1. OFDM 解调 (FFT)
   └─→ 频域符号

2. 信道估计
   └─→ 每个子载波的信道响应

3. DCM 解映射 ←── 关键步骤
   └─→ 合并两个子载波的数据

4. 解调 (Demodulation)
   └─→ 软判决或硬判决

5. 解交织
   └─→ 恢复原始顺序

6. 解码 (Decoding)
   └─→ BCC 或 LDPC 解码

7. 解扰
   └─→ 最终数据
```

### 3.4 DCM 合并算法

```python
# DCM 合并算法（接收机）
def dcm_combining(symbol_1, symbol_2, h_1, h_2, noise_var):
    """
    合并两个子载波上的重复符号
    
    参数:
    symbol_1, symbol_2: 接收到的符号
    h_1, h_2: 两个子载波的信道响应
    noise_var: 噪声方差
    
    返回:
    combined: 合并后的符号估计
    """
    # 最大比合并 (MRC)
    weight_1 = conj(h_1) / noise_var
    weight_2 = conj(h_2) / noise_var
    
    combined = (weight_1 * symbol_1 + weight_2 * symbol_2) / \
               (abs(h_1)**2 + abs(h_2)**2)
    
    return combined

# 或使用选择合并
def dcm_selection(symbol_1, symbol_2, h_1, h_2):
    """
    选择信噪比更高的子载波
    """
    snr_1 = abs(h_1)**2
    snr_2 = abs(h_2)**2
    
    if snr_1 >= snr_2:
        return symbol_1 / h_1
    else:
        return symbol_2 / h_2
```

## 4. DCM 应用场景

### 4.1 适用场景

```
DCM 最适合的场景:

1. 高干扰环境
   - 多 AP 共存
   - 邻频干扰
   - 蓝牙共存

2. 低 SNR 场景
   - 远距离传输
   - 穿墙通信
   - 多径衰落

3. 高可靠性需求
   - 关键控制帧
   - 低延迟应用
   - 工业物联网

4. 低速率应用
   - 物联网设备
   - 传感器数据
   - 心跳包
```

### 4.2 性能对比

```
DCM vs 非 DCM 性能对比 (MCS 0, BPSK):

SNR (dB)   非 DCM PER    DCM PER
  0          50%          30%
  5          20%          5%
  10         5%           0.5%
  15         1%           0.01%

结论:
- DCM 在低 SNR 时显著降低 PER
- 高 SNR 时差距缩小
- 代价是速率减半
```

### 4.3 DCM 与 MCS 选择

```
MCS 选择策略:

如果 SNR < 10 dB:
  → 使用 DCM + MCS 0/1

如果 SNR 在 10-20 dB:
  → 可选 DCM + MCS 3/4 或 非 DCM + MCS 2-4

如果 SNR > 20 dB:
  → 使用 非 DCM + MCS 5+

实际选择还需考虑:
- 信道带宽
- 干扰情况
- 延迟要求
- 功耗限制
```
