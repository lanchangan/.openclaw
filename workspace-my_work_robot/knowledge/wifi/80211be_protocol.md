## 1. 协议概述

### 1.1 WiFi 7 简介

IEEE 802.11be (WiFi 7) 是下一代超高速无线局域网标准，目标是实现 **30 Gbps** 吞吐量。

**核心特性**:
- **320 MHz 带宽**: 带宽翻倍
- **4096-QAM**: 更高调制效率 (12 bits/symbol)
- **MRU**: 多 RU 组合
- **Multi-Link**: 多链路并发
- **Preamble Puncturing**: 前导码打孔
- **16 空间流**: 理论最大配置

**性能指标**:
| 参数 | 数值 |
|------|------|
| 最大带宽 | 320 MHz |
| 最大空间流 | 16 (理论) |
| 最高调制 | 4096-QAM |
| 最大速率 | 46 Gbps (理论) |
| 目标速率 | 30 Gbps (实际) |
| 频段 | 2.4 GHz, 5 GHz, 6 GHz |

### 1.2 与 WiFi 6 对比

| 特性 | WiFi 6 (802.11ax) | WiFi 7 (802.11be) |
|------|-------------------|-------------------|
| 最大带宽 | 160 MHz | 320 MHz |
| 最大调制 | 1024-QAM | 4096-QAM |
| RU 机制 | 单 RU | MRU (多 RU) |
| 最大空间流 | 8 | 16 |
| 多链路 | 无 | MLO |
| 峰值速率 | 9.6 Gbps | 46 Gbps |

## 2. PHY 层新特性

### 2.1 EHT-MCS 表

| MCS 索引 | 调制方式 | 码率 | NBPSC | 相对增益 |
|---------|---------|------|-------|---------|
| 0-11 | 同 WiFi 6 | - | - | - |
| 12 | 4096-QAM | 3/4 | 12 | +20% |
| 13 | 4096-QAM | 5/6 | 12 | +20% |

### 2.2 U-SIG 字段

WiFi 7 引入 U-SIG (Universal SIG) 字段，用于前向兼容：

```
U-SIG 结构 (80 bit = 2 个符号):
┌─────────────────────────────────────────────────────────────┐
│ U-SIG Symbol 1 (40 bit)                                     │
├─────────────────────────────────────────────────────────────┤
│ PHY Version Identifier (3 bits)  │ 识别 EHT/未来版本        │
│ BSS Color (6 bits)               │ BSS 着色                 │
│ TXOP (7 bits)                    │ 传输机会                 │
│ EHT-SIG Symbol Count (4 bits)    │ EHT-SIG 符号数           │
│ CRC (4 bits)                     │ 校验                     │
│ Tail (6 bits)                    │ 尾比特                   │
│ Reserved (10 bits)               │ 保留                     │
├─────────────────────────────────────────────────────────────┤
│ U-SIG Symbol 2 (40 bit)                                     │
├─────────────────────────────────────────────────────────────┤
│ PPDU Type (2 bits)               │ PPDU 类型                 │
│ UL/DL (1 bit)                    │ 上行/下行                 │
│ MCS (4 bits)                     │ 调制编码方案              │
│ Bandwidth (3 bits)               │ 带宽编码                  │
│ ...                              │                          │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 带宽编码

| 带宽编码 | 带宽 | 实际带宽 |
|---------|------|---------|
| 000 | 20 MHz | 20 MHz |
| 001 | 40 MHz | 40 MHz |
| 010 | 80 MHz | 80 MHz |
| 011 | 160 MHz | 160 MHz |
| 100 | 320 MHz-1 | 320 MHz (频率配置1) |
| 101 | 320 MHz-2 | 320 MHz (频率配置2) |

## 3. 320 MHz 带宽

### 3.1 信道配置

WiFi 7 支持 320 MHz 带宽，主要在 6 GHz 频段实现：

```
6 GHz 信道配置:

中心频率 = 5950 + 5 × n (MHz)

320 MHz 信道:
┌─────────────────────────────────────────────────────┐
│ 320 MHz-1: 中心频率 5985 MHz (n=7)                  │
│ 范围: 5825 - 6145 MHz                              │
├─────────────────────────────────────────────────────┤
│ 320 MHz-2: 中心频率 6105 MHz (n=31)                 │
│ 范围: 5945 - 6265 MHz                              │
└─────────────────────────────────────────────────────┘
```

### 3.2 子载波配置

| 带宽 | 子载波总数 | 数据子载波 | 导频子载波 |
|------|-----------|-----------|-----------|
| 320 MHz | 4096 | 3920 | 80 |

### 3.3 打孔模式

WiFi 7 支持 Primary 20/40/80/160 MHz 打孔：

```
320 MHz 打孔示例:

原始终态 (无打孔):
┌─────────────────────────────────────────────────┐
│P20│ S20│ S40│ S80│ S160                        │
└─────────────────────────────────────────────────┘

80 MHz 打孔:
┌─────────────────────────────────────────────────┐
│P20│ S20│    打孔   │ S160                        │
└─────────────────────────────────────────────────┘

160 MHz 打孔:
┌─────────────────────────────────────────────────┐
│P20│ S20│ S40│ S80│       打孔                  │
└─────────────────────────────────────────────────┘
```

## 4. MRU 机制

### 4.1 MRU 定义

MRU (Multiple RU) 允许组合相邻 RU，提高资源利用率：

**MRU 类型**:
| MRU 类型 | 组成 | 子载波数 |
|---------|------|---------|
| MRU-52+26 | 52-tone + 26-tone | 78 |
| MRU-106+26 | 106-tone + 26-tone | 132 |
| MRU-106+52 | 106-tone + 52-tone | 158 |
| MRU-242+26 | 242-tone + 26-tone | 268 |
| MRU-484+242 | 484-tone + 242-tone | 726 |
| MRU-996+484 | 996-tone + 484-tone | 1480 |
| MRU-996+484+242 | 996 + 484 + 242 | 1722 |
| MRU-2x996+484 | 2×996 + 484 | 2476 |

### 4.2 MRU 组合规则

```
MRU 组合约束:

1. 只能组合相邻 RU
2. 小 RU 必须在前面 (低频)
3. 组合必须符合标准定义

有效示例:
┌─────────┬──────┐
│ 106-tone│26-tone│  → MRU-106+26 ✓
└─────────┴──────┘

无效示例:
┌──────┬─────────┐
│26-tone│ 106-tone│  → 不符合规则 ✗
└──────┴─────────┘
```

### 4.3 MRU 分配示例

```
320 MHz MRU 分配示例:

方式1: 1 × MRU-2x996+484 (大用户)
┌─────────────────────────────────────────────────────────────┐
│              MRU-2x996+484 (2476 tones)                     │
└─────────────────────────────────────────────────────────────┘

方式2: 混合 RU + MRU
┌───────────┬───────────┬────────────────────────────────────┐
│ MRU-106+26│  996-tone │       MRU-996+484                  │
└───────────┴───────────┴────────────────────────────────────┘
```

## 5. 4096-QAM 调制

### 5.1 星座图

4096-QAM 每符号携带 12 bits：

```
4096-QAM 星座特性:

I/Q 范围: [-63.5, +63.5]
星座点数: 64 × 64 = 4096
每符号比特: log2(4096) = 12 bits

相比 1024-QAM:
- 比特数: 12 vs 10 (+20% 效率)
- 星座点: 4096 vs 1024 (4倍密度)
- SNR 要求: 更高 (约 +4 dB)
```

### 5.2 SNR 要求

| MCS | 调制 | 码率 | 典型 SNR 要求 |
|-----|------|------|--------------|
| 10 | 1024-QAM | 3/4 | ~30 dB |
| 11 | 1024-QAM | 5/6 | ~32 dB |
| 12 | 4096-QAM | 3/4 | ~35 dB |
| 13 | 4096-QAM | 5/6 | ~37 dB |

### 5.3 EVM 要求

| MCS | EVM 要求 (RMS) |
|-----|---------------|
| 0-9 | 同 WiFi 6 |
| 10 | -35 dB |
| 11 | -37 dB |
| 12 | -40 dB |
| 13 | -42 dB |

## 6. Multi-Link 操作

### 6.1 MLD 概念

Multi-Link Device (MLD) 支持同时在多个频段操作：

```
AP MLD 结构:

┌─────────────────────────────────────┐
│           AP MLD                    │
├──────────┬──────────┬───────────────┤
│ Link 1   │ Link 2   │ Link 3        │
│ 2.4 GHz  │ 5 GHz    │ 6 GHz         │
└──────────┴──────────┴───────────────┘

STA MLD 结构:

┌─────────────────────────────────────┐
│          STA MLD                    │
├──────────┬──────────┬───────────────┤
│ Link 1   │ Link 2   │ Link 3        │
│ 2.4 GHz  │ 5 GHz    │ 6 GHz         │
└──────────┴──────────┴───────────────┘
```

### 6.2 链路模式

| 模式 | 描述 | 特点 |
|------|------|------|
| STR | Simultaneous Transmit and Receive | 最高吞吐 |
| NSTR | Non-Simultaneous TR | 减少干扰 |
| EMLSR | Enhanced Multi-Link Single Radio | 单无线电多链路 |

### 6.3 流量映射

```
流量到链路的映射:

┌──────────┐     ┌─────────────────┐
│  应用层   │────→│  流量分类器      │
└──────────┘     └────────┬────────┘
                          │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
    ┌─────────┐     ┌─────────┐     ┌─────────┐
    │ Link 1  │     │ Link 2  │     │ Link 3  │
    │ 低延迟  │     │ 中速    │     │ 高吞吐  │
    └─────────┘     └─────────┘     └─────────┘
```

## 7. 验证要点

### 7.1 TX 验证矩阵

| 测试项 | 描述 | 覆盖范围 |
|-------|------|---------|
| EHT PPDU 生成 | 4 种 PPDU 类型 | 格式正确性 |
| 320 MHz 发送 | 全带宽测试 | 频谱模板 |
| MRU 分配 | 所有 MRU 类型 | 组合规则 |
| 4096-QAM | MCS 12-13 | EVM 性能 |
| 打孔模式 | 80/160 MHz 打孔 | 打孔正确性 |
| MLO 发送 | 多链路并发 | 链路协调 |

### 7.2 RX 验证矩阵

| 测试项 | 描述 | 覆盖范围 |
|-------|------|---------|
| U-SIG 解析 | 版本识别 | 前向兼容 |
| 320 MHz 接收 | 全带宽 | 频偏估计 |
| MRU 接收 | 多 RU 组合 | RU 分离 |
| 4096-QAM 解调 | MCS 12-13 | 解调性能 |
| 打孔检测 | 识别打孔模式 | 打孔指示 |
| MLO 接收 | 多链路同步 | 链路聚合 |

### 7.3 关键检查点

**U-SIG 验证**:
- [ ] PHY Version 正确解码
- [ ] BSS Color 正确提取
- [ ] 带宽编码正确
- [ ] CRC 校验通过

**MRU 验证**:
- [ ] MRU 类型识别
- [ ] RU 组合顺序
- [ ] 子载波分配正确
- [ ] 边界对齐

**320 MHz 验证**:
- [ ] 两种频率配置
- [ ] 打孔模式正确
- [ ] 频谱模板符合
- [ ] 频偏估计精度

**MLO 验证**:
- [ ] 链路发现
- [ ] 链路选择
- [ ] 流量映射
- [ ] 无缝切换
