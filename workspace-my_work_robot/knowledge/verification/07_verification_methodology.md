## 2. 验证策略

### 2.1 验证方法选择

```
┌─────────────────────────────────────────────────────────────┐
│                    验证方法比较                              │
├──────────────┬──────────────┬──────────────┬───────────────┤
│ 方法         │ 优点         │ 缺点         │ 适用场景      │
├──────────────┼──────────────┼──────────────┼───────────────┤
│ 定向测试     │ 简单直接     │ 覆盖有限     │ 基础功能验证  │
│              │ 易于调试     │ 工作量大     │ 协议合规检查  │
├──────────────┼──────────────┼──────────────┼───────────────┤
│ 随机测试     │ 覆盖广       │ 难复现       │ 回归测试      │
│              │ 效率高       │ 需要检查器   │ 压力测试      │
├──────────────┼──────────────┼──────────────┼───────────────┤
│ 形式验证     │ 完全覆盖     │ 状态爆炸     │ 关键路径验证  │
│              │ 数学证明     │ 工具限制     │ 协议检查      │
├──────────────┼──────────────┼──────────────┼───────────────┤
│ 断言验证     │ 实时检查     │ 需要经验     │ 协议检查      │
│              │ 定位快       │ 覆盖有限     │ 内部逻辑检查  │
└──────────────┴──────────────┴──────────────┴───────────────┘
```

### 3.2 验证组件复用

```systemverilog
// 可复用的 Agent
class apb_agent extends uvm_agent;
    // 参数化配置
    uvm_active_passive_enum is_active = UVM_ACTIVE;
    int timeout_cycles = 1000;
    
    // 复用配置
    function void build_phase(uvm_phase phase);
        // 从 config_db 获取配置
        uvm_config_db #(uvm_active_passive_enum)::get(
            this, "", "is_active", is_active);
    endfunction
endclass

// 不同环境复用
class Env1 extends uvm_env;
    apb_agent apb;  // 复用 APB Agent
endclass

class Env2 extends uvm_env;
    apb_agent apb;  // 同样复用
    axi_agent axi;  // 加上 AXI Agent
endclass
```

## 4. 回归测试

### 4.1 回归测试策略

```
┌─────────────────────────────────────────────────────────────┐
│                    回归测试层次                              │
├─────────────────────────────────────────────────────────────┤
│  每日回归 (Nightly Regression)                              │
│  ├── 冒烟测试 (快速)                                        │
│  ├── 关键功能测试                                           │
│  └── 目标: 快速反馈                                         │
├─────────────────────────────────────────────────────────────┤
│  周回归 (Weekly Regression)                                 │
│  ├── 完整功能测试                                           │
│  ├── 覆盖率检查                                             │
│  └── 目标: 全面验证                                         │
├─────────────────────────────────────────────────────────────┤
│  里程碑回归 (Milestone Regression)                          │
│  ├── 全量回归                                               │
│  ├── 形式验证                                               │
│  ├── 门级仿真                                               │
│  └── 目标: 流片前确认                                       │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 回归测试管理

```tcl
# Makefile 示例
.PHONY: smoke full regression

smoke:
	vcs -full64 -sverilog +v2k -timescale=1ns/1ps \
		+define+SMOKE_TEST \
		-f filelist.f \
		-top top_tb

full:
	vcs -full64 -sverilog +v2k -timescale=1ns/1ps \
		+define+FULL_TEST \
		-f filelist.f \
		-top top_tb

regression:
	./run_regression.pl -testlist test.list -parallel 8
```

### 4.3 测试结果分析

```systemverilog
// 测试结果统计
class TestResult;
    string test_name;
    int    pass_count;
    int    fail_count;
    real   coverage;
    time   simulation_time;
    
    function void print_report();
        $display("Test: %s", test_name);
        $display("  Pass: %0d, Fail: %0d", pass_count, fail_count);
        $display("  Coverage: %.2f%%", coverage);
        $display("  Sim Time: %t", simulation_time);
    endfunction
endclass
```

## 6. 最佳实践

### 6.1 代码规范

```systemverilog
// 命名规范
// 类名: 大驼峰
class MyTransaction;
// 变量: 小写 + 下划线
bit [31:0] my_data;
// 常量: 全大写
parameter MAX_COUNT = 100;
// 宏: 全大写
`define MAX_DELAY 10

// 注释规范
// 单行注释
/*
 * 多行注释
 */

/**
 * 文档注释
 * @param name 参数说明
 * @return 返回值说明
 */
function int my_function(int name);
endfunction
```
