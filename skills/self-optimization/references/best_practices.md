# 自我优化最佳实践

## 记忆管理最佳实践

### 1. 记忆分类原则
- **长期记忆（MEMORY.md）**：只保存重要的、需要长期记住的信息：
  - 用户的偏好和习惯
  - 重要的配置信息和凭据
  - 经验教训和最佳实践
  - 长期任务和目标
- **短期记忆（memory/YYYY-MM-DD.md）**：保存日常操作日志、临时信息、任务细节
- **归档记忆（memory/archive/）**：超过7天的短期记忆自动归档

### 2. 记忆清理规则
- 每天自动清理超过30天的归档记忆
- 删除重复的信息和已经处理过的TODO项
- 敏感信息（密码、密钥等）加密存储

## 流程优化最佳实践

### 1. 任务执行评估标准
- 执行时间是否在预期范围内
- 是否有可以自动化的重复步骤
- 错误率是否超过阈值（>10%）
- 用户满意度是否达标

### 2. 优化优先级
1. 高频重复任务 → 优先自动化
2. 高错误率任务 → 优先改进流程
3. 长时间运行任务 → 优先优化性能
4. 用户反馈不好的任务 → 优先调整交互方式

## 技能学习最佳实践

### 1. 技能获取原则
- 优先从官方clawhub仓库获取技能
- 安装前检查技能的评分和下载量
- 新技能安装后必须进行测试验证
- 只安装实际需要的技能，避免冗余

### 2. 技能更新策略
- 每周检查一次已安装技能的更新
- 重要安全更新立即安装
- 功能更新先在测试环境验证再更新
- 更新后记录变更日志

## 自我诊断最佳实践

### 1. 诊断频率
- 每天执行一次完整诊断
- 任务失败后立即执行针对性诊断
- 系统升级后执行全量诊断
- 长时间运行（>7天）后执行诊断

### 2. 问题处理流程
1. 警告级问题 → 记录并计划修复
2. 严重级问题 → 立即修复并通知用户
3. 未知问题 → 收集日志并提交反馈
4. 修复完成后重新执行诊断验证
