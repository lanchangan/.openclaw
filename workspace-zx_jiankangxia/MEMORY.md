# 全局记忆 - 小红书发布相关问题&解决方法
## 2026-03-28 发布桥本检查笔记遇到的问题&解决方案
### 问题1：Unicode编码报错
- 现象：执行发布命令时报错 `UnicodeEncodeError: 'gbk' codec can't encode character '\u2705' in position 0`
- 原因：xiaohongshu-mcp的xhs_client.py脚本里使用了✅、❌等emoji字符，Windows系统默认gbk编码不支持这类字符
- 解决方案：修改`C:\Users\LWS\.openclaw\skills\xiaohongshu-mcp\scripts\xhs_client.py`文件，移除所有emoji字符，换成纯文字提示（已完成修改）

### 问题2：文案换行失效，显示多余`\n`
- 现象：发布的笔记里换行没有生效，反而显示了大量`\n`转义字符
- 原因：传入文案时使用了转义的`\n`，没有用真实换行
- 解决方案：后续发布笔记时，传入的文案内容直接使用真实换行，不要用`\n`转义字符

### 问题3：发布后无法获取post ID
- 现象：发布成功后返回 `Post ID: Unknown`，无法定位具体笔记
- 原因：MCP接口返回数据中没有post_id字段，或者脚本解析失败
- 解决方案：目前无法远程删除/管理已发布的笔记，发布重复内容后需要手动在小红书APP内删除，后续优化脚本解析逻辑

## 后续发布注意事项
1. 发布前检查文案没有多余转义字符，换行使用真实换行
2. 相同内容不要重复执行发布命令，避免生成重复笔记
3. 发布完成后手动到小红书APP确认内容排版、图片是否正常
